# aipp - C++ Starter

A minimal C++20 starter project: CMake + presets + Atlas strong types +
DocTest + RapidCheck. Use it as the seed for new C++ projects in this
style.

## Atlas Strong Types

This project uses the Atlas Strong Type Generator. **All coding and review
agents must read `docs/atlas-for-ai-assistants.md`** for proper Atlas usage
patterns.

Key rules:
- Never use raw types where strong types exist.
- Never use `atlas::undress()` in domain logic -- only at serialization
  boundaries.
- Add features to type definitions instead of unwrapping.
- All public interfaces (functions, members, etc) must use semantic strong
  types.

Atlas is expected to be installed on the system as the `atlas` executable
(found via `find_program`). It is **not** fetched via `FetchContent` in
this project.

## Code Style

- **Class names**: `PascalCase`
- **Functions**: `snake_case`
- **Private members**: trailing underscore (`member_`)
- **Template params**: `PascalCaseWithTrailingT` (e.g. `ArgTs`)
- **Concepts**: `PascalCaseWithTrailingC` (e.g. `AtlasTypeC`)
- **Header guards**: UUID-based: `AIPP_<UUID>`
- **NVI pattern**: Virtual functions are always private, prefixed with `do_`
- **Column limit**: 80
- **Namespace**: `aipp` (and `aipp::testing` for test-support code)

## Build

```bash
cmake --preset debug && cmake --build --preset debug && ctest --preset debug
```

`debug` and `release` are aliases for the clang variants. The full preset
matrix is debug/release × {gcc, gcc-asan, clang}. ASan is enabled by
default for clang and explicit `-asan` presets.

## Build Options

- `AIPP_BUILD_TESTS` (default: `ON`) - Build unit tests
- `AIPP_SANITIZE` (default: `ON`) - Enable AddressSanitizer
- `AIPP_CACHE_LINE_SIZE` - Override hardware cache-line detection
- `Atlas_EXECUTABLE` - Override `find_program` discovery of `atlas`

## Dependencies

External tool (must already be installed):
- **Atlas** -- located via `find_program(atlas)`. Override with
  `-DAtlas_EXECUTABLE=/path/to/atlas`.

Fetched via `FetchContent` when `AIPP_BUILD_TESTS=ON`:
- **DocTest** -- unit testing framework
- **RapidCheck** -- property-based testing, with doctest integration
  (`rc::doctest::check`)

## Adding a New Strong Type

1. Add a `[class TypeName]` block to `src/types.atlas`.
2. Build -- `atlas` regenerates `src/types_gen.hpp` automatically.
3. Use the type via `#include "types_gen.hpp"` and reference it as
   `aipp::TypeName`.
4. To extract the underlying value (sparingly!) use the ADL-found
   `atlas_value_for(x)`; do **not** add a `value()` accessor.

See `docs/atlas-for-ai-assistants.md` for the full usage guide.
